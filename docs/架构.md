# **4.** **数据库设计（****SQLite****）**

## **4.1 Schema** **总览**

使用单一 SQLite 文件（默认路径 ~/.context-engine/{repo_hash}.db），包含 4 张核心表：

| **表名**   | **行数量级**             | **核心用途**               |
| ---------- | ------------------------ | -------------------------- |
| files      | 与仓库文件数相当         | 文件元信息、哈希、解析状态 |
| symbols    | 每文件 		10–200 行 | 函数/类/方法的完整信息     |
| call_edges | 每函数 		2–10 行   | 调用关系有向图             |
| index_meta | 固定 < 		10 行     | 全局索引状态、版本信息     |

# **5.** **解析层设计（****tree-sitter****）**

## **5.1** **支持语言与** **Grammar** **包**

| **语言**   | **tree-sitter 		Grammar** **包** | **提取目标**                                                 |
| ---------- | -------------------------------------- | ------------------------------------------------------------ |
| Python     | tree-sitter-python                     | function_definition, class_definition, 		decorated_definition |
| TypeScript | tree-sitter-typescript                 | function_declaration, method_definition, 		arrow_function, class_declaration |
| JavaScript | tree-sitter-javascript                 | 同 		TypeScript，去掉类型节点                          |
| Go         | tree-sitter-go                         | function_declaration, 		method_declaration, type_declaration |
| Rust       | tree-sitter-rust                       | function_item, impl_item, struct_item, 		enum_item     |
| Java       | tree-sitter-java                       | method_declaration, class_declaration, 		interface_declaration |


 

## **5.2** **解析流程**

| # 		engine/parser.py  核心逻辑示意                     |
| ------------------------------------------------------------ |
|                                                              |
| import 		tree_sitter_python as tspython                |
| from 		tree_sitter import Language, Parser, Node       |
|                                                              |
| class 		SymbolExtractor:                               |
| """从单个源文件提取所有 		Symbol 		信息"""       |
|                                                              |
| FUNCTION_NODES 		= {                                   |
| "python": 		    ["function_definition"],               |
| "typescript": 		["function_declaration", "method_definition", |
| "arrow_function"],                                           |
| "go": 		        ["function_declaration", "method_declaration"], |
| }                                                            |
|                                                              |
| def 		extract(self, file_path: str, lang: str) -> list[SymbolRecord]: |
| source 		= Path(file_path).read_bytes()                |
| tree 		  = self._parsers[lang].parse(source)           |
| return 		self._walk(tree.root_node, source, lang)      |
|                                                              |
| def 		_walk(self, node: Node, src: bytes, lang: str,   |
| parent: 		str \| None = None) -> list[SymbolRecord]:   |
| results 		= []                                         |
| if 		node.type in self.FUNCTION_NODES[lang]:           |
| rec 		= self._extract_symbol(node, src, parent)        |
| results.append(rec)                                          |
| # 		递归：提取嵌套函数（parent 		设为当前函数名） |
| for 		child in node.children:                          |
| results 		+= self._walk(child, src, lang, rec.name)    |
| else:                                                        |
| for 		child in node.children:                          |
| results 		+= self._walk(child, src, lang, parent)      |
| return 		results                                       |


 

## **5.3** **调用关系提取策略**

在 AST 遍历过程中同步提取调用关系，避免二次扫描。针对不同语言使用对应的调用节点类型：

| **语言**   | **调用节点类型**                                | **处理要点**                                           |
| ---------- | ----------------------------------------------- | ------------------------------------------------------ |
| Python     | call → function → identifier                    | 区分方法调用 		(attr) 和函数调用 		(name)  |
| TypeScript | call_expression → function → 		identifier | 处理链式调用、可选链 		?.                        |
| Go         | call_expression → function                      | 区分包调用 		pkg.Func 和本地调用                 |
| 通用       | —                                               | 只记录 		callee_name，跨文件解析延迟到索引后处理 |

# **7.** **索引构建与增量更新**

## **7.1** **全量索引流程**

| **步骤**           | **操作**                                                | **关键实现点**                                  |
| ------------------ | ------------------------------------------------------- | ----------------------------------------------- |
| 1. 文件发现        | 递归遍历仓库，过滤 		.gitignore 和配置的排除规则  | 使用 		pathspec 库解析 		.gitignore |
| 2. 语言检测        | 根据文件扩展名映射到 		tree-sitter Grammar        | 维护扩展名到语言的映射表                        |
| 3. 哈希计算        | 对文件内容计算 		SHA256，与 		DB 中记录对比 | 跳过未变更文件，增量时重用                      |
| 4. AST 解析        | 调用 		SymbolExtractor 		提取所有符号       | 并行解析（ThreadPoolExecutor）                  |
| 5. 写入 		DB | 批量 		INSERT symbols 和 		call_edges       | 使用事务批量提交（每 		1000 条）          |
| 6. 跨文件解析      | 遍历 		call_edges，填充 		callee_id 外键    | 按符号名建内存映射表加速查找                    |
| 7. 更新元信息      | 写入 		index_meta.last_full_index                 | 标记索引完成                                    |


 

## **7.2** **增量更新流程**

| # 		engine/indexer.py  增量更新核心逻辑                |
| ------------------------------------------------------------ |
|                                                              |
| def 		incremental_update(self, changed_files: list[str]) -> 		IndexResult: |
| """对变更文件执行增量重索引"""                               |
| with 		self.db.transaction():                          |
| for 		path in changed_files:                           |
| if 		not path_exists(path):                            |
| # 		文件删除：级联删除（ON 		DELETE CASCADE 自动处理 		symbols/call_edges） |
| self.db.execute("DELETE 		FROM files WHERE path = ?", [path]) |
| continue                                                     |
|                                                              |
| new_hash 		= sha256(read_bytes(path))                  |
| old_record 		= self.db.get_file(path)                  |
|                                                              |
| if 		old_record and old_record.content_hash == new_hash: |
| continue 		 # 内容未变化（如仅权限位变更），跳过       |
|                                                              |
| # 		删除旧符号（外键级联），重新解析                   |
| if 		old_record:                                       |
| self.db.execute("DELETE 		FROM files WHERE id = ?", [old_record.id]) |
|                                                              |
| # 		重新解析并插入                                     |
| symbols 		= self.parser.extract(path, detect_lang(path)) |
| file_id 		= self.db.insert_file(path, new_hash)        |
| self.db.bulk_insert_symbols(file_id, 		symbols)        |
|                                                              |
| # 		重新解析跨文件调用关系（仅限受影响文件）           |
| self._resolve_call_edges(changed_files)                      |


 

## **7.3** **文件监听器（****Watcher****）**

| # 		engine/watcher.py                                  |
| ------------------------------------------------------------ |
| from 		watchdog.observers import Observer              |
| from 		watchdog.events   import FileSystemEventHandler |
| import 		threading, time                               |
|                                                              |
| class 		RepoWatcher(FileSystemEventHandler):           |
| DEBOUNCE_SECONDS 		= 0.5   # 防抖：500ms 		内多次变更合并处理 |
|                                                              |
| def 		__init__(self, indexer, repo_root):              |
| self._indexer 		  = indexer                            |
| self._pending 		  = set()        # 待处理的变更文件    |
| self._timer 		    = None                               |
| self._lock 		     = threading.Lock()                   |
|                                                              |
| def 		on_modified(self, event):                        |
| if 		not event.is_directory and is_code_file(event.src_path): |
| self._schedule(event.src_path)                               |
|                                                              |
| def 		_schedule(self, path):                           |
| with 		self._lock:                                     |
| self._pending.add(path)                                      |
| if 		self._timer:                                      |
| self._timer.cancel()                                         |
| self._timer 		= threading.Timer(                       |
| self.DEBOUNCE_SECONDS, 		self._flush                   |
| )                                                            |
| self._timer.start()                                          |
|                                                              |
| def 		_flush(self):                                    |
| with 		self._lock:                                     |
| files 		= list(self._pending)                          |
| self._pending.clear()                                        |
| self._indexer.incremental_update(files) 		 # 触发增量更新 |


 

# **8.** **查询引擎设计**

## **8.1** **精确查询（****get_symbol****）**

| # 		engine/query.py                                    |
| ------------------------------------------------------------ |
|                                                              |
| def 		get_symbol(self, name: str, file: str \| None, kind: str \| None). |
| -> 		SymbolRecord \| None:                             |
| sql 		= """                                            |
| SELECT 		s.*, f.path, f.lang                           |
| FROM 		  symbols s                                     |
| JOIN 		  files   f ON f.id = s.file_id                 |
| WHERE 		 s.name = ?                                    |
| """                                                          |
| params 		= [name]                                      |
| if 		file:                                             |
| sql 		+= " AND f.path LIKE ?"                          |
| params.append(f"%{file}%")                                   |
| if 		kind:                                             |
| sql 		+= " AND s.kind = ?"                             |
| params.append(kind)                                          |
| sql 		+= " LIMIT 1"                                    |
| row 		= self.db.fetchone(sql, params)                  |
| return 		SymbolRecord.from_row(row) if row else None   |


 

## **8.2** **全文搜索（****search_code****）**

| # 		利用 		SQLite 		FTS5 全文索引实现高效语义搜索 |
| ------------------------------------------------------------ |
|                                                              |
| def 		search(self, query: str, limit: int = 10,        |
| lang: 		str \| None = None) -> list[SymbolRecord]:     |
| sql 		= """                                            |
| SELECT 		s.*, f.path, f.lang,                          |
| fts.rank 		AS score                                    |
| FROM 		  symbols_fts fts                               |
| JOIN 		  symbols s ON s.id = fts.rowid                 |
| JOIN 		  files   f ON f.id = s.file_id                 |
| WHERE 		 symbols_fts MATCH ?                           |
| """                                                          |
| params 		= [query]                                     |
| if 		lang:                                             |
| sql 		+= " AND f.lang = ?"                             |
| params.append(lang)                                          |
| sql 		+= " ORDER BY score LIMIT ?"                     |
| params.append(limit)                                         |
| rows 		= self.db.fetchall(sql, params)                 |
| return 		[SymbolRecord.from_row(r) for r in rows]      |


 

## **8.3** **调用图查询（****get_callers / get_callees****）**

| # 		递归 		CTE 		实现多跳调用关系查询（depth 		控制深度） |
| ------------------------------------------------------------ |
|                                                              |
| GET_CALLERS_SQL 		= """                                |
| WITH 		RECURSIVE callers(id, name, depth) AS (         |
| -- 		Base case: 直接调用者                             |
| SELECT 		s.id, s.name, 1                               |
| FROM 		  call_edges e                                  |
| JOIN 		  symbols    s ON s.id = e.caller_id            |
| WHERE 		 e.callee_id = (SELECT id FROM symbols WHERE name = ? LIMIT 1) |
| UNION 		ALL                                            |
| -- 		Recursive case: 间接调用者                        |
| SELECT 		s.id, s.name, c.depth + 1                     |
| FROM 		  call_edges  e                                 |
| JOIN 		  symbols     s ON s.id = e.caller_id           |
| JOIN 		  callers     c ON c.id = e.callee_id           |
| WHERE 		 c.depth < ?    -- 限制深度                    |
| )                                                            |
| SELECT 		DISTINCT s.*, f.path, callers.depth           |
| FROM 		  callers                                       |
| JOIN 		  symbols s ON s.id = callers.id                |
| JOIN 		  files   f ON f.id = s.file_id                 |
| ORDER 		 BY callers.depth, s.name                      |
| """                                                          |

# **14.** **已知限制与后续扩展**

## **14.1 Phase 1–4** **的已知限制**

- ​	**动态语言精度：**Python 的动态类型导致部分间接调用无法静态分析（如通过 getattr 的调用）
- ​	**跨语言仓库：**同一函数调用跨语言（如 Python 调用 C 扩展）暂不追踪
- ​	**宏****/****装饰器：**高度宏展开的代码（Rust proc_macro）符号提取可能不完整
- ​	**全文搜索质量：**FTS5 基于词法匹配，中文代码注释需配合 jieba 分词（可选扩展）


 

## **14.2 Phase 5+** **扩展方向**

- ​	**语义向量搜索：**引入 ollama/nomic-embed-text 生成 embedding，结合 sqlite-vss 实现语义相似度搜索
- ​	**类型推断集成：**对 TypeScript 集成 tsc --noEmit 的类型信息，提升调用图精度
- ​	**符号重命名追踪：**结合 git log 追踪符号历史变更
- ​	**多仓库联邦查询：**支持 monorepo 或跨仓库依赖的调用图合并
- ​	**IDE** **插件：**实现 LSP 兼容接口，在 VS Code / JetBrains 中直接使用
